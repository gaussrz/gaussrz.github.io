Arora & Barak Chapter 15 - Average Case Complexity

gaussrz
2022-04-04
computational complexity, theoretical computer science

\[ \newcommand{\B}{\mathbf} \newcommand{\R}{\mathrm} \newcommand{\distNP}{\mathrm{dist}\mathbf{NP}} \newcommand{\distP}{\mathrm{dist}\mathbf{P}} \newcommand{\Pr}{\mathbf{Pr}} \newcommand{\E}{\mathbf{E}} \] Many average case algorithms focus on problems on random graphs. The average is in the sense of the randomness of the graphs. Many \(\mathbf{NP}\)-complete problems are easy in the random case, such as \(\R{CLIQUE}\), \(\R{3-COLOR}\), and \(\R{INDEPENDENT\; SET}\). The average case complexity is polynomial or nearly polynomial.

Note that although \(\B{NP}\)-complete problems are equivalent in the worst case complexity, they may differ in the average case complexity. The point is that the worst case reduction no longer works in the average case, since Karp reduction does not preserve the distribution of the input. This chapter is to present Levin's theory that formalize these problems.

Comments: 1. Should that be the case that the Karp reduction, or mapping between input spaces can exhibit a hard part or easy part of a problem? 2. Levin's theory, of course, should take care the mappings between distributions. I'm wondering in what way did Levin state them: the independence of the mappings (that says nothing actually) or finer ways? The construction of average case complete problems might be based on Turing machines or self-directional problems, since it might be easy to find a hardcore in a regular problem.

Distributional Problems

The following definition is natural:

Definition (distributional problems) A distributional problem is a pair \(\langle L, \mathcal D\rangle\), where \(L\) is a decision problem, \(\mathcal D\) is a distribution on \(\{0, 1\}^*\).

Interestingly, the distribution \(\mathcal D\) is defined an all lengths of inputs, rather than a separate distribution on each length of input. One of the reason I see is the definition of average case complexity, that a problem is still averagely good if all algorithms perform poorly on several certain lengths of inputs.

Comments: If there exists 'hard' inputs, it's meaningless to discuss arbitrary distribution of inputs, since by making the distribution accumulates among 'bad' inputs, the average case complexity can be arbitrarily close to the worst case complexity. So it's important to discuss the existence of hard inputs.

To describe the 'real world distribution', we define two 'good' classes of distributions: - Polynomial time computable distributions: distributions that the CDFs is polynomial time computable, the CDF is defined by the lexicographical order. - Polynomial time samplable distributions: distributions that can be sampled in polynomial time. The first \(\subset\) the second, and if \(\B P \neq \B P ^\B{\#P}\), the first is a proper subset of the second.

An immediate question: is the definition of computable distributions identical to all orders? Or at least the orders that compatible with input lengths?

\(\distNP\) and Its Complete Problems

Polynomial Time on Average

This chapter focus on \(\B{NP}\) problems with polynomial time computable distribution of inputs, i.e. \[ \distNP:=\{\langle L, \mathcal D \rangle\ | L\in \B{NP}, \mathcal D\text{ polynomial time computable}\}. \]

The first thing to do is to define an analogous \(\B P\) class. To my surprise, the definition is based on the expectation rather than the analogy of 'convergence in probability'.

Definition (\(\distP\)) A problem \(\langle L, \mathcal D\rangle\in\distNP\) is in \(\distP\), if there exists an algorithm \(A\) with running time \(t(x)\) on the input \(x\) such that \[ \E \left[ \frac{t(x)^{1/c}}{|x|} \right] = c_1 \] for some constants \(c, c_1\).

The reason to define the \(\distP\) in this manner is to ensure the invariability under polynomial time reduction. Suppose there is a reduction that squares the computational time (in the worst case, this is definitely a valid Karp reduction), the reduced problem is still in \(\distP\).

It's a routine task to bound the probability of quick calculation by the expectation by Chebyshev's inequality, the detailed result is omitted here.

Reductions

This is probably the most expected part in this chapter. The definition is quite natural: > Definition (reduction): > A distributional problem \(\langle L_1, \mathcal D_1\rangle\), reduces to a problem \(\langle L_2, \mathcal D_2\rangle\), denoted by \(\langle L_1, \mathcal D_1\rangle\le\langle L_2, \mathcal D_2\rangle\), if there exists a polynomial-time computable mapping \(f\) such that: > 1. \(x\in L_1 \iff f(x) \in L_2\); > 2. \(\forall x: |f(x)| = \Omega(|x|^\epsilon)\); > 3. \(\forall x : \Pr_{f\circ\mathcal D_1}[x] \le c_1|x|^c Pr_{\mathcal D_2}[x]\), > where \(\epsilon\) and \(c, c_1\) are positive constants.

The first two is regular reduction definition, and the third one rules out the condition that the mapping \(f\) maps a high proportion of inputs to the hard part of \(L_2\), since we want 'if \(L_2\) is computable, then \(L_1\) is computable'. The following theorem says the above definition satisfies our requirements.

Theorem If \(\langle L_1, \mathcal D_1\rangle\le\langle L_2, \mathcal D_2\rangle\) and \(\langle L_2, \mathcal D_2\rangle\) has an algorithm that is polynomial on average, then there is a polynomial-average algorithm that computes \(\langle L_2, \mathcal D_2\rangle\).

A Complete Problem in \(\distNP\)

In this section, we construct a complete problem in \(\distNP\), which is named after Levin, albeit this problem is quite artificial. Just as remarked, this construction is based on certification Turing machines.

Definition (the universal distributional problem) The problem \(\langle U, \mathcal D \rangle\) takes inputs in the form of \(\langle M, x, 1^t, 1^l\rangle\), where \(M\) is a Turing machine. The input is in \(U\) iff there exists \(y\) such that \(|y| \le l\) and \(M(x, y)\) accepts in \(t\) steps. \(\mathcal D\) is an uniform distribution such that \[ \Pr_\mathcal D [\langle M, x, 1^t, 1^l \rangle ] = \frac{1} {|M|(|M|+1)2^{-|M|}} \frac1{|x|(|x|+1)2^{-|x|}} \frac1{(s+t)(s+t+1)}. \]

This definition is just the analogy of \(\R{TMSAT}\), equipped with a certain distribution. The reason for the last to inputs is to insure the problem in \(\distNP\), i.e. polynomial time solvable by a nondeterministic Turing machine.

The rest work is to construct a valid mapping according to the definition of reduction. The construction is probably the most beautiful part in this chapter, which is written below: > Definition (peak elimination mapping) > For a distribution \(\mathcal D\), define a mapping \(h\) as follows: > 1. if \(\Pr_\mathcal D[x]\le 2^{-|x|}\), then \(h(x)=x\); > 2. if \(\Pr_\mathcal D [x]>2^{-|x|}\), then \(h(x)\) is the longest common prefix of \(\mu_\mathcal D(x)\) and \(\mu_\mathcal D (x-1)\), > where \(\mu_\mathcal D\) is the CDF of the distribution \(\mathcal D\). > Then, define the peak elimination mapping \(g\): > $$ g(x) = { \[\begin{array}{ll} 1h(x),&\Pr[x]\le 2^{-|x|}\\ 0h(x),&\Pr[x]> 2^{-|x|} \end{array}\]

.

$$

Intuitively, \(g\) changes the lengths of inputs according to their probability, and make the mapping unique by choosing the common prefix as the new representation. It's all about the properties of the longest common prefix. \(g\) is carefully made to construct a reduction to \(\langle U, \mathcal D\rangle\), that is \[ x \mapsto\mathcal \langle M', z:=g(x), 1^{|x|^c+|x|}, 1^{|x|^c+|x|^d}\rangle, \] where \(M'\) is a modification of the certification Turing machine \(M\) of \(L\), \(M\) certificates whether \(x\in L\) with a witness \(y\), \(|y|\le |x|^c\) in time \(|x|^c\). \(M'\) takes a witness \((x, y)\) of length \(|x|+|x|^c\) as input, and checks whether \(M(x, y)\) accepts in time \(|x|^c\) and \(g(x)=z\). \(|x|^d\) is the overhead of computing \(g\).

This is a valid reduction, so here is Levin's theorem: > Theorem (Levin) > \(\langle U, \mathcal D \rangle\) is \(\distNP\)-complete.

Another Definition of Polynomial Average Time

This definition is in the 'convergence in probability' manner. > Definition (heuristic polynomial time) > An polynomial time algorithm \(A\) for the \(\distNP\) problem \(\langle L, \mathcal D\rangle\) is heuristic polynomial time, if > \[ \sum_{A(x)\neq \chi_L(x)} \Pr_D[x] p(|x|) < \infty \] > for any polynomial \(p\).

This definition worth careful inspection. Intuitively, the probability that \(A\) fails descends rapidly with \(|x|\). This definition is incompatible with the definition of \(\distP\), they cannot derive each other. But it's easy to see the definition of reduction is still valid for heuristic polynomial time algorithms.

Polynomial Time Samplability

There is a result by Impagliazzo and Levin that if a problem is \(\distNP\)-complete, then it is also \(\langle \B{NP}, \B P\text{-samplable}\rangle\)-complete.
